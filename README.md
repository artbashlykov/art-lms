# ART LMS

WordPress-плагин: мини-LMS с автовыдачей цифровых продуктов и приёмом платежей (ЮKassa, ЮMoney, Prodamus, Plisio).

**Версия:** 2.16.2  
**Требования:** WordPress 6.0+, PHP 7.4+

## Установка из репозитория

1. Склонируйте репозиторий в `wp-content/plugins/art-lms`.
2. Для локальной разработки скопируйте `gitignore.template` в `.gitignore` (файл не входит в релиз).
3. Активируйте плагин в админке WordPress.
4. Настройте страницы, платежи и материалы в разделе **ART LMS**.

## Обновления (GitHub Releases)

Плагин проверяет обновления через [Plugin Update Checker](https://github.com/YahnisElsts/plugin-update-checker) из публичного репозитория [artbashlykov/art-lms](https://github.com/artbashlykov/art-lms). На странице **Плагины** доступны ссылка **Проверить обновления** и стандартное уведомление WordPress при новой версии; автообновление включается там же, если хостинг это поддерживает.

Токен GitHub на сайте не нужен. Для приватного форка в `wp-config.php`:

```php
define( 'ART_LMS_GITHUB_TOKEN', 'your-github-token' );
```

## Ограничения и важные моменты

- **Доступ к материалам** выдаётся через учётную запись WordPress после оплаты, а не через отдельную «гостевую» сессию.
- **Файлы из медиатеки**, вставленные в материалы, отдаются через защищённый маршрут `art-lms-media`. На части хостингов прямой URL `/wp-content/uploads/...` может обходить PHP — после обновления пересохраните материалы, чтобы зарегистрировать вложения.
- **Возвраты и chargeback** автоматически не обрабатываются: отзовите доступ вручную, сменив статус заказа на «Возврат» или «Отменён».
- **Plisio:** статус `mismatch` учитывается в пределах допустимого отклонения, настроенного в кабинете Plisio. Если в callback есть `source_amount`, сумма сверяется с заказом.
- **Тестовый шлюз** не подтверждает оплату автоматически — отключайте его на боевом сайте.
